### TECHNICAL SPECIFICATION: ROGUE SURVIVAL

---

### 1. ARCHITECTURE MAPPING (RAG INTEGRATION)

*   **Game State Management**: Custom module (not in RAG) handles `Menu`, `Playing`, `Pause`, `Rules`, `GameOver`, `Level-Up` states.
*   **Player & Entity Base Class**: `sprite_manager.GameSprite` serves as the foundation for Player, Enemies, Projectiles, Items.
*   **Object Pooling**: `object_pool.ObjectPool` is explicitly used for:
    *   Projectiles (e.g., player bullets, enemy projectiles).
    *   Experience Gems (collectibles).
    *   Regular Enemy instances (for wave management and rapid respawn).
*   **Tilemap Generation & Environment Structure**: `tile_map.MazeManager` is utilized for procedural level layout generation, defining `TILE_WALL` and `TILE_PATH` elements which map to `COLLISION_TYPE: "solid"` for obstacles and navigable areas respectively.
*   **Player Camera System**: `camera_player_center.CameraScrollGroup` implements the dynamic camera, ensuring the player is smoothly centered on screen with Y-sort rendering.
*   **Collision Detection**: `collision.CollisionManager` provides static utility methods (`apply_sprite_vs_group`, `apply_group_vs_group`) for all in-game collision logic.
*   **Monster AI Pathfinding (Swarm)**: `dijkstra.FlowFieldPathfinder` is used for general enemies to efficiently navigate around terrain obstacles and flow towards the player, optimizing for "dozens of monsters".
*   **Monster AI Pathfinding (Elite/Boss)**: `Astar.AStarPathfinder` is reserved for elite enemies or bosses requiring more precise, calculated paths around obstacles.
*   **Spatial Grid Optimization**: A conceptual `SpatialGrid` module (not provided in RAG) is critical for:
    *   Optimizing broad-phase collision detection (e.g., `ProjectileGroup` vs `EnemyGroup`).
    *   Efficiently querying entities within a specific area (e.g., finding nearby enemies for targeting, collecting experience gems).
    *   Reducing the number of collision checks from O(N^2) to O(N) or O(N log N) in dense environments.

### 4. COLLISION MATRIX

| Initiating Entity     | Target Entity        | Resulting Action(s)                                                                  |
| :-------------------- | :------------------- | :----------------------------------------------------------------------------------- |
| **Player**            | Regular Enemy        | Player takes `ENEMY_DAMAGE`. Enemy applies `ENEMY_DAMAGE_COOLDOWN` to player.        |
|                       | Elite Enemy          | Player takes `ELITE_DAMAGE`. Enemy applies `ELITE_DAMAGE_COOLDOWN` to player.        |
|                       | Player Projectile    | NO COLLISION.                                                                        |
|                       | Experience Gem       | Player collects `XP_VALUE`. Gem `kill()`s (returns to ObjectPool).                   |
|                       | Wall Obstacle (`solid`)| Player movement is stopped.                                                        |
|                       | Platform (`one_way`) | Player can move horizontally. Player can jump up through, lands on top. Cannot fall through from top. |
| **Regular Enemy**     | Player Projectile    | Enemy takes `PROJECTILE_DAMAGE`. Projectile `kill()`s (returns to ObjectPool).       |
|                       | Experience Gem       | NO COLLISION.                                                                        |
|                       | Wall Obstacle (`solid`)| Enemy movement is stopped/pathfinds around.                                        |
|                       | Platform (`one_way`) | Enemy moves horizontally on top. Pathfinding avoids moving under/through.            |
| **Elite Enemy**       | Player Projectile    | Elite Enemy takes `PROJECTILE_DAMAGE`. Projectile `kill()`s (returns to ObjectPool). |
|                       | Experience Gem       | NO COLLISION.                                                                        |

|                       | Platform (`one_way`) | Elite Enemy moves horizontally on top. Pathfinding avoids moving under/through.      |

|                       | Platform (`one_way`) | Projectile `kill()`s (returns to ObjectPool).                                        |

### 5. GAMEPLAY LOGIC & PHYSICS DATA

*   **Win Condition**: A level is cleared when `enemy_count == 0` for the current wave, triggering transition to `Level-Up Screen`.
*   **Loss Condition**: The game ends (`Game Over` state) when `player_hp == 0`.
*   **Physics Properties**:
    *   `COLLISION_TYPE: "solid"`: Used for `wall_tile.png` and other impassable environmental entities. Entities attempting to move into a "solid" object will have their movement vector adjusted to prevent overlap.
    *   `COLLISION_TYPE: "one_way"`: Used for `platform_one_way.png` and similar platforms. Player can move freely on top. Player can move from below through its bottom edge. Downward movement from above will register collision from the top edge.

### 6. DYNAMIC IMAGE SCALING (MATHEMATICAL RULE)

*   **Base Asset Resolution**: ALL generated visual assets are assumed to be 1024x1024 pixels.
*   **Screen Resolution**: Game operates on a 1280x720 screen.
*   **`IMAGE_SCALE` Calculation**: A float value for each entity determines its rendered size on screen, calculated as `(Desired_Pixel_Size / Base_Asset_Resolution)`.
    *   Player/Enemy entities (`[sprite]player.png`, `[sprite]enemy_basic.png`): Approximately 80px (1024 * 0.08 = 81.92px). `IMAGE_SCALE: 0.08`.
    *   Projectiles/Items (`[sprite]projectile_bullet.png`, `[sprite]item_xp_gem.png`): Approximately 20px (1024 * 0.02 = 20.48px). `IMAGE_SCALE: 0.02`.
    *   UI Buttons (`[sprite]button_game_start.png`, etc.): Approximately 150px (1024 * 0.15 = 153.6px). `IMAGE_SCALE: 0.15`.
    *   Backgrounds/Map Tiles (`[background]game_map_tile.png`): Approximately 1280px (1024 * 1.25 = 1280px) to ensure horizontal screen fill. `IMAGE_SCALE: 1.25`.
